Cline은 AWS Bedrock 및 Google Cloud Vertex와 같은 주요 클라우드 제공업체를 지원합니다. 팀에서 현재 사용 중인 제공업체가 무엇이든 적합하며, Cline의 기능을 활용하기 위해 제공업체를 변경할 필요는 없습니다.

이 문서에서는 조직에서 클라우드 기반 프론티어 모델을 사용한다고 가정합니다. 클라우드 추론 제공업체는 최첨단 기능과 필요에 가장 적합한 모델을 선택할 수 있는 유연성을 제공합니다.

매우 민감한 데이터 처리, 일관된 짧은 대기 시간 응답이 필요한 애플리케이션 또는 엄격한 데이터 주권 요구 사항 준수를 포함한 특정 시나리오에서는 로컬 모델 사용이 필요할 수 있습니다. 팀에서 로컬 모델을 활용해야 하는 경우 Cline과 함께 [로컬 모델 실행](/running-models-locally/read-me-first.mdx)을 참조하십시오.

#### VPC 엔드포인트 설정

팀 데이터를 보호하기 위해 Cline은 VPC(Virtual Private Cloud) 엔드포인트를 지원하여 데이터와 AWS Bedrock 간에 프라이빗 연결을 생성합니다. AWS VPC는 공용 IP 주소, 네트워크 게이트웨이 또는 복잡한 방화벽 규칙이 필요 없도록 하여 보안을 강화합니다. 기본적으로 공용 인터넷을 완전히 우회하는 데이터용 프라이빗 고속도로를 만드는 것입니다. AWS의 프라이빗 네트워크 내에서 트래픽을 유지함으로써 팀은 AWS Bedrock 또는 사용자 지정 API와 같은 서비스에 액세스할 때 지연 시간이 짧고 예측 가능한 성능의 이점도 누릴 수 있습니다. 기밀 정보를 다루거나 의료 또는 금융과 같이 규제가 심한 산업에서 운영하는 경우 VPC는 클라우드 서비스의 접근성과 프라이빗 인프라의 보안 간의 완벽한 균형을 제공합니다.

---

1. VPC 엔드포인트 생성에 대한 [AWS 가이드](https://docs.aws.amazon.com/bedrock/latest/userguide/vpc-interface-endpoints.html)를 참조하십시오. 이 문서에는 사전 요구 사항이 명시되어 있으며 VPC 엔드포인트 생성에 사용되는 구문이 설명되어 있습니다.
2. AWS 콘솔에서 [VPC 엔드포인트 생성](https://docs.aws.amazon.com/vpc/latest/privatelink/create-interface-endpoint.html#create-interface-endpoint-aws) 지침을 따르십시오. 아래 이미지는 위에 링크된 AWS 가이드의 4단계와 5단계에 해당합니다.

<Frame>
	<img src="https://storage.googleapis.com/cline_public_images/docs/assets/vpc-console.png" alt="VPC 콘솔" />
</Frame>

3. VPC 엔드포인트의 IP 주소를 기록하고 Cline의 설정 메뉴를 연 다음 API 공급자 드롭다운에서 `AWS Bedrock`을 선택합니다.
4. `사용자 지정 VPC 엔드포인트 사용` 확인란을 클릭하고 VPC 엔드포인트의 IP 주소를 입력합니다.
