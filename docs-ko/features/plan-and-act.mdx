계획 및 실행 모드는 구현 전에 신중한 계획을 강조하는 구조화된 AI 개발에 대한 Cline의 접근 방식을 나타냅니다. 이 이중 모드 시스템은 개발자가 반복 시간을 줄이면서 유지 관리가 용이하고 정확한 코드를 만드는 데 도움이 됩니다.

#### 계획 모드: 먼저 생각하기

계획 모드는 사용자와 Cline이 무엇을 만들려고 하는지, 어떻게 만들 것인지 파악하는 곳입니다. 이 모드에서 Cline은 다음을 수행합니다.

-   컨텍스트를 이해하기 위해 전체 코드베이스를 읽을 수 있습니다.
-   파일을 변경하지 않습니다.
-   요구 사항을 이해하고 전략을 수립하는 데 중점을 둡니다.
-   코드 한 줄을 작성하기 전에 잠재적인 문제를 식별하는 데 도움이 됩니다.

#### 실행 모드: 만들기

계획이 수립되면 실행 모드로 전환합니다. 이제 Cline은 다음을 수행합니다.

-   사용 가능한 모든 빌딩 기능을 갖추고 있습니다.
-   코드베이스를 변경할 수 있습니다.
-   계획 세션의 모든 내용을 여전히 기억합니다.
-   함께 작업한 전략을 실행합니다.

### 워크플로 가이드

새로운 기능을 작업하거나 복잡한 버그를 수정할 때 저에게 효과적인 방법은 다음과 같습니다.

1. 계획 모드에서 시작하여 Cline에게 만들고 싶은 것을 알립니다.
2. Cline은 관련 파일을 보면서 코드베이스를 탐색하는 데 도움을 줍니다.
3. 함께 예외적인 경우와 잠재적인 문제를 고려하여 최상의 접근 방식을 파악합니다.
4. 계획에 확신이 들면 실행 모드로 전환합니다.
5. Cline은 계획에 따라 솔루션을 구현합니다.

#### 1. 계획 모드로 시작

모든 중요한 개발 작업은 계획 모드에서 시작합니다.

이 모드에서는 다음을 수행합니다.

#### 2. 실행 모드로 전환

명확한 계획이 수립되면 실행 모드로 전환합니다.

<Frame>
	<img src="https://storage.googleapis.com/cline_public_images/docs/assets/switching-to-act.gif" alt="실행 모드로 전환" />
</Frame>

실행 모드를 사용하면 Cline이 다음을 수행할 수 있습니다.

-   합의된 계획에 따라 실행합니다.
-   코드베이스를 변경합니다.
-   계획 단계의 컨텍스트를 유지합니다.

#### 3. 필요에 따라 반복

복잡한 프로젝트에는 종종 여러 계획-실행 주기가 필요합니다.

-   예상치 못한 복잡성에 직면하면 계획 모드로 돌아갑니다.
-   솔루션 구현에 실행 모드를 사용합니다.
-   품질을 보장하면서 개발 추진력을 유지합니다.

### 모범 사례

#### 계획 단계

1. 요구 사항을 포괄적으로 작성합니다.
2. 관련 컨텍스트를 미리 공유합니다.
3. Cline이 읽지 않은 경우 관련 파일을 가리킵니다.
4. 구현 전에 접근 방식을 검증합니다.

#### 구현 단계

1. 수립된 계획을 따릅니다.
2. 목표에 대한 진행 상황을 모니터링합니다.
3. 변경 사항과 그 영향을 추적합니다.
4. 중요한 결정을 문서화합니다.

### 고급 사용자 팁

#### 계획 향상

-   구현 전에 예외적인 경우를 탐색하기 위해 계획 모드를 사용합니다.
-   예상치 못한 복잡성에 직면하면 계획으로 다시 전환합니다.
-   가정을 조기에 검증하기 위해 [파일 읽기](/features/at-mentions/file-mentions)를 활용합니다.
-   나중에 참조할 수 있도록 Cline이 계획의 마크다운 파일을 작성하도록 합니다.

### 일반적인 패턴

#### 각 모드를 사용하는 경우

계획 모드는 다음과 같은 경우에 가장 효과적이라는 것을 알았습니다.

-   접근 방식이 명확하지 않은 새로운 작업을 시작할 때
-   무엇이 잘못되었는지 확신할 수 없는 까다로운 문제를 디버깅할 때
-   코드베이스의 여러 부분에 영향을 미칠 아키텍처 결정을 내릴 때
-   복잡한 워크플로 또는 기능을 이해하려고 할 때

그리고 실행 모드는 다음에 적합합니다.

-   이미 계획한 솔루션을 구현할 때
-   접근 방식이 명확한 일상적인 변경을 할 때
-   코드베이스에서 확립된 패턴을 따를 때
-   테스트를 실행하고 사소한 조정을 할 때
