## 작동 방식

기본적으로 Cline은 파일 읽기 또는 쓰기를 포함하여 도구를 호출하기 전에 사용자 권한을 요청합니다.

Cline이 묻지 않고 작업을 수행하도록 허용하려면 해당 도구에 대한 자동 승인 권한을 설정할 수 있습니다.

## 권한 옵션

-   **프로젝트 파일 읽기**

    -   Cline이 현재 작업 공간 내의 파일을 묻지 않고 읽도록 허용합니다.
    -   **모든 파일 읽기**
        -   작업 공간 외부의 파일(시스템 파일, 구성 파일 등)에 대한 읽기 권한을 확장합니다.

-   **프로젝트 파일 편집**

    -   Cline이 확인 없이 현재 작업 공간 내의 파일을 수정하도록 허용합니다.
    -   **모든 파일 편집**
        -   작업 공간 외부의 파일에 대한 수정 권한을 확장합니다.

-   **안전한 명령 실행**

    -   모델이 비파괴적이라고 판단하는 터미널 명령 실행을 허용합니다.
    -   **모든 명령 실행**
        -   묻지 않고 모든 터미널 명령 실행을 허용합니다.

-   **브라우저 사용**

    -   Cline이 브라우저 도구를 사용하여 웹 콘텐츠를 가져오도록 허용합니다.

-   **MCP 서버 사용**

    -   확장된 기능을 위해 MCP 서버에 연결하고 사용하도록 허용합니다.

-   **최대 요청 수**
    -   Cline이 사용자 입력을 요구하기 전에 수행할 수 있는 연속적인 자동화된 작업 수를 설정합니다.

## 모범 사례

개인적으로 자동 편집을 비활성화하는 것을 선호합니다. 모든 단계에서 변경 사항을 검토할 수 있는 기회를 제공하기 때문입니다.

대부분의 심각한 개발 워크플로의 경우 다음으로 시작하는 것이 좋습니다.

-   프로젝트 파일에 대한 읽기 액세스 자동 승인
-   합리적인 최대 요청 제한 설정(10-20)

이렇게 하면 지속적인 중단 없이 코드베이스를 탐색할 수 있는 충분한 자유를 Cline에 제공하는 동시에 편집이나 잠재적으로 파괴적인 작업에 대한 권한은 여전히 필요합니다.

특정 프로젝트에 대한 Cline의 기능에 대한 신뢰를 구축함에 따라 편안함 수준에 맞게 권한을 점차적으로 늘릴 수 있습니다.

필요에 따라 언제든지 이러한 설정을 조정할 수 있다는 점을 기억하십시오. 중요한 프로덕션 작업에 대한 권한을 강화하거나 프로토타이핑 및 탐색 시 권한을 완화하십시오.

빠른 "별표" 작업을 사용하여 이동하면서 자동 승인된 선택 항목을 빠르게 켜고 끌 수도 있습니다.
