---
title: "새 규칙 명령어"
sidebarTitle: "/newrule"
---

`/newrule`은 Cline에게 선호하는 작업 방식을 가르칠 수 있는 슬래시 명령어입니다. 프로젝트 지원 시 Cline이 어떻게 행동해야 하는지에 대한 지속적인 지침 역할을 하는 `.clinerules` 디렉터리에 마크다운 파일을 만듭니다.

Cline이 항상 따를 하우스 규칙을 설정하는 것으로 생각하면 매 대화마다 기본 설정을 반복할 필요가 없습니다.

#### `/newrule` 슬래시 명령어 사용

Cline이 특정 지침을 일관되게 따르도록 하려면 다음을 수행합니다.

-   채팅에 `/newrule`을 입력합니다.
-   Cline은 다음에 대한 기본 설정을 질문하여 구조화된 규칙 파일을 만드는 데 도움을 줄 것입니다.
    -   커뮤니케이션 스타일(자세함 vs. 간결함)
    -   개발 워크플로
    -   코딩 표준
    -   프로젝트 컨텍스트
    -   기타 특정 지침
-   규칙 파일이 생성되기 전에 검토합니다.
-   승인되면 Cline은 향후 대화에 자동으로 로드될 `.clinerules` 디렉터리에 마크다운 파일을 만듭니다.

#### 예시

모든 새 작업에서 동일한 지침을 반복하는 데 지쳤을 때 `/newrule`을 사용했습니다. React 구성 요소 구성 방식, 사용할 테스트 라이브러리, 심지어 선호하는 변수 이름 지정 스타일에 대한 특정 기본 설정이 있었습니다.

매번 이러한 기본 설정을 입력하는 대신 `/newrule`을 사용하고 Cline과 협력하여 상세한 규칙 파일을 만들었습니다. 코드 구성에서 클래스 구성 요소보다 함수형 구성 요소를 선호하는 것까지 모든 것을 다루는 마크다운 파일을 만들었습니다.

이제 React 프로젝트에 대해 Cline과 채팅할 때마다 상기시켜 줄 필요 없이 이러한 지침을 자동으로 따릅니다. 가장 좋은 점은 서로 다른 프로젝트에 대해 서로 다른 규칙 파일을 만들 수 있으므로 Cline이 작업 중인 모든 코드베이스에 적응한다는 것입니다.

#### 영감

개발을 더 원활하게 하기 위해 `/newrule`을 사용하는 방법은 다음과 같습니다.

-   특정 아키텍처 패턴과 라이브러리 기본 설정이 있는 각 주요 프로젝트에 대한 규칙 파일을 만들었으므로 Cline은 항상 기존 코드베이스와 일치하는 코드를 생성합니다.

-   팀의 공유 프로젝트에는 누가 Cline을 사용하든 일관된 코드 스타일과 문서화 관행을 보장하는 공통 규칙 파일이 있습니다.

-   레거시 코드로 작업할 때 이전 시스템의 특징과 제약 조건을 Cline에 상기시키는 규칙 파일을 만들었으므로 잘 통합되지 않는 최신 접근 방식을 제안하지 않습니다.

-   사이드 프로젝트에는 두 칸 들여쓰기, 모든 곳에 화살표 함수 사용, 정확한 폴더 구조 요구 사항 등 모든 독단적인 기본 설정이 포함된 개인 규칙 파일도 있습니다.
