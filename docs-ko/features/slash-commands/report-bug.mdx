title: "버그 보고 명령어"
sidebarTitle: "/reportbug"
---

`/reportbug`는 Cline에서 이상한 문제에 직면했을 때 절대적으로 생명의 은인입니다. 버그 보고서에 대해 GitHub가 원하는 모든 세부 정보를 기억해야 하는 대신 이 명령어는 Cline을 개인 버그 보고 도우미로 바꿔줍니다.

적절한 버그 보고서에 필요한 모든 정보를 수집하는 과정을 안내한 다음 올바른 서식과 시스템 세부 정보가 모두 포함된 상태로 GitHub 문제 페이지로 바로 보냅니다.

#### `/reportbug` 슬래시 명령어 사용

뭔가 이상해 보이는 문제가 발생하면 다음을 수행합니다.

-   채팅에 `/reportbug`를 입력하기만 하면 됩니다.
-   Cline이 필요한 모든 세부 정보를 안내해 줄 것입니다.
    -   문제를 설명하는 간단한 제목
    -   실제로 일어난 일과 예상했던 일
    -   버그를 재현하는 단계
    -   본 관련 출력이나 오류
    -   문제를 해결하는 데 도움이 될 수 있는 추가 컨텍스트
-   제출하기 전에 모든 내용을 검토할 수 있습니다.
-   승인하면 모든 정보와 자동 시스템 세부 정보가 포함된 완벽하게 서식이 지정된 GitHub 문제가 열립니다.

#### 예시

지난주에 큰 파일을 읽을 때 Cline이 계속 시간 초과되는 이상한 버그에 직면했습니다. GitHub 템플릿 필드를 모두 기억하려고 하는 대신 `/reportbug`를 입력했고 Cline이 전체 과정을 안내해 주었습니다.

무엇을 하려고 했는지, 대신 무슨 일이 일어났는지, 문제로 이어진 정확한 단계를 물었습니다. 가장 좋았던 점은 OS 버전, Cline 버전 및 개발자가 필요로 하는 모든 기술적 세부 정보가 자동으로 포함되었다는 것입니다.

몇 초 후 해당 정보를 직접 찾을 필요 없이 제대로 서식이 지정된 GitHub 문제가 생성되었습니다.
