Cline은 OpenAI API 표준과 호환되는 API를 제공하는 광범위한 AI 모델 공급자를 지원합니다. 이를 통해 OpenAI 이외의 공급자 모델을 사용하면서도 익숙한 API 인터페이스를 활용할 수 있습니다. 여기에는 다음과 같은 공급자가 포함됩니다.

-   Ollama 및 LM Studio와 같은 도구를 통해 실행되는 **로컬 모델**(각각의 섹션에서 다룸).
-   Perplexity, Together AI, Anyscale 등과 같은 **클라우드 공급자**.
-   OpenAI 호환 API 엔드포인트를 제공하는 **기타 모든 공급자**.

이 문서는 공식 OpenAI API 이외의 공급자 설정에 중점을 둡니다(공식 OpenAI API에는 자체 [전용 구성 페이지](/provider-config/openai)가 있음).

### 일반 구성

OpenAI 호환 공급자를 Cline과 함께 사용하는 핵심은 다음과 같은 기본 설정을 구성하는 것입니다.

1.  **기본 URL:** 공급자별 API 엔드포인트입니다. `https://api.openai.com/v1`이 아닙니다(해당 URL은 공식 OpenAI API용임).
2.  **API 키:** 선택한 공급자로부터 받는 비밀 키입니다.
3.  **모델 ID:** 사용하려는 모델의 특정 이름 또는 식별자입니다.

Cline 설정 패널(⚙️ 아이콘 클릭)에서 이러한 설정을 찾을 수 있습니다.

-   **API 공급자:** "OpenAI 호환"을 선택합니다.
-   **기본 URL:** 선택한 공급자가 제공한 기본 URL을 입력합니다. **이것은 중요한 단계입니다.**
-   **API 키:** 공급자의 API 키를 입력합니다.
-   **모델:** 모델 ID를 선택하거나 입력합니다.
-   **모델 구성:** 이 섹션에서는 다음과 같은 모델의 고급 매개변수를 사용자 지정할 수 있습니다.
    -   최대 출력 토큰
    -   컨텍스트 창 크기
    -   이미지 지원 기능
    -   컴퓨터 사용(예: 도구/함수 호출 기능이 있는 모델의 경우)
    -   입력 가격(토큰/백만 토큰당)
    -   출력 가격(토큰/백만 토큰당)

### 지원되는 모델 (OpenAI 네이티브 엔드포인트용)

"OpenAI 호환" 공급자 유형을 사용하면 다양한 엔드포인트에 연결할 수 있지만 공식 OpenAI API(또는 이를 정확히 미러링하는 엔드포인트)에 직접 연결하는 경우 Cline은 소스 코드의 `openAiNativeModels` 정의를 기반으로 다음 모델 ID를 인식합니다.

-   `o3-mini`
-   `o3-mini-high`
-   `o3-mini-low`
-   `o1`
-   `o1-preview`
-   `o1-mini`
-   `gpt-4.5-preview`
-   `gpt-4o`
-   `gpt-4o-mini`

**참고:** 다른 OpenAI 호환 공급자(예: Together AI, Anyscale 등)를 사용하는 경우 사용 가능한 모델 ID가 다릅니다. 지원되는 모델 이름 및 고유한 구성 세부 정보에 대해서는 항상 특정 공급자의 설명서를 참조하십시오.

### Cline의 v0 (Vercel SDK):

-   v0으로 작업하는 개발자의 경우 [AI SDK 설명서](https://vercel.com/docs/v0/cline)는 다양한 모델(대부분 OpenAI 호환)을 통합하기 위한 귀중한 통찰력과 예제를 제공합니다. 이는 Vercel에 배포되거나 통합된 서비스와 함께 Cline을 사용할 때 호출을 구성하고 구성을 관리하는 방법을 이해하는 데 유용한 리소스가 될 수 있습니다.

-   v0은 OpenAI 호환 공급자와 함께 Cline에서 사용할 수 있습니다.

-   ### 빠른 시작

-   1. OpenAI 호환 공급자가 선택된 상태에서 기본 URL을 https://api.v0.dev/v1로 설정합니다.
-   2. v0 API 키를 붙여넣습니다.
-   3. 모델 ID를 v0-1.0-md로 설정합니다.
-   4. 확인을 클릭하여 연결을 확인합니다.

### 문제 해결

-   **"잘못된 API 키":** API 키를 올바르게 입력했는지, 올바른 공급자용인지 다시 확인합니다.
-   **"모델을 찾을 수 없음":** 선택한 공급자에 유효한 모델 ID를 사용하고 있고 지정된 기본 URL에서 사용할 수 있는지 확인합니다.
-   **연결 오류:** 기본 URL이 올바른지, 공급자의 API에 컴퓨터에서 액세스할 수 있는지, 방화벽이나 네트워크 문제가 없는지 확인합니다.
-   **예상치 못한 결과:** 예상치 못한 출력이 나오는 경우 다른 모델을 시도하거나 모든 구성 매개변수를 다시 확인합니다.

OpenAI 호환 공급자를 사용하면 더 광범위한 AI 모델과 함께 Cline의 유연성을 활용할 수 있습니다. 가장 정확하고 최신 정보는 항상 공급자의 설명서를 참조하십시오.
