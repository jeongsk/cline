OpenRouter는 단일 API를 통해 다양한 공급자의 다양한 언어 모델에 대한 액세스를 제공하는 AI 플랫폼입니다. 이를 통해 설정을 단순화하고 다양한 모델을 쉽게 실험할 수 있습니다.

**웹사이트:** [https://openrouter.ai/](https://openrouter.ai/)

### API 키 받기

1.  **가입/로그인:** [OpenRouter 웹사이트](https://openrouter.ai/)로 이동합니다. Google 또는 GitHub 계정으로 로그인합니다.
2.  **API 키 받기:** [키 페이지](https://openrouter.ai/keys)로 이동합니다. API 키가 나열되어 있어야 합니다. 그렇지 않은 경우 새 키를 만듭니다.
3.  **키 복사:** API 키를 복사합니다.

### 지원되는 모델

OpenRouter는 많고 증가하는 수의 모델을 지원합니다. Cline은 사용 가능한 모델 목록을 자동으로 가져옵니다. 전체 및 최신 목록은 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 참조하십시오.

### Cline의 구성

1.  **Cline 설정 열기:** Cline 패널에서 설정 아이콘(⚙️)을 클릭합니다.
2.  **공급자 선택:** "API 공급자" 드롭다운에서 "OpenRouter"를 선택합니다.
3.  **API 키 입력:** OpenRouter API 키를 "OpenRouter API 키" 필드에 붙여넣습니다.
4.  **모델 선택:** "모델" 드롭다운에서 원하는 모델을 선택합니다.
5.  **(선택 사항) 사용자 지정 기본 URL:** OpenRouter API에 사용자 지정 기본 URL을 사용해야 하는 경우 "사용자 지정 기본 URL 사용"을 선택하고 URL을 입력합니다. 대부분의 사용자는 이 필드를 비워 둡니다.

### 지원되는 변환

OpenRouter는 모델의 최대 컨텍스트 크기를 초과하는 프롬프트를 지원하기 위해 [선택적 "중간 출력" 메시지 변환](https://openrouter.ai/docs/features/message-transforms)을 제공합니다. "프롬프트 및 메시지 체인을 컨텍스트 크기로 압축" 확인란을 선택하여 활성화할 수 있습니다.

### 팁 및 참고 사항

-   **모델 선택:** OpenRouter는 광범위한 모델을 제공합니다. 필요에 가장 적합한 모델을 찾기 위해 실험해 보십시오.
-   **가격 책정:** OpenRouter는 기본 모델의 가격 책정을 기준으로 요금을 청구합니다. 자세한 내용은 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 참조하십시오.
-   **프롬프트 캐싱:**
    -   OpenRouter는 이를 지원하는 기본 모델에 캐싱 요청을 전달합니다. 캐싱을 제공하는 모델을 확인하려면 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 확인하십시오.
    -   대부분의 모델에서 모델 자체가 지원하는 경우 캐싱이 자동으로 활성화되어야 합니다(Requesty 작동 방식과 유사).
    -   **OpenRouter를 통한 Gemini 모델 예외:** OpenRouter를 통해 액세스할 때 Google의 캐싱 메커니즘에서 때때로 관찰되는 잠재적인 응답 지연으로 인해 _특히 Gemini 모델_의 경우 수동 활성화 단계가 필요합니다.
    -   OpenRouter를 통해 **Gemini 모델**을 사용하는 경우 해당 모델에 대한 캐싱을 활성화하려면 공급자 설정에서 "프롬프트 캐싱 활성화" 확인란을 **수동으로 선택해야 합니다**. 이 확인란은 임시 해결 방법 역할을 합니다. OpenRouter의 비 Gemini 모델의 경우 이 확인란은 캐싱에 필요하지 않습니다.
