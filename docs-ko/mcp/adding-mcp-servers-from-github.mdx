---
title: "GitHub에서 MCP 서버 추가"
---

이 가이드는 Cline을 사용하여 GitHub 리포지토리에서 기존 MCP 서버를 빌드하는 방법에 대한 단계별 연습을 제공합니다.

## **MCP 서버 찾기**

온라인에서 MCP 서버를 찾을 수 있는 여러 곳이 있습니다.

-   **Cline은 MCP 서버를 목록에 자동으로 추가한 다음 편집할 수 있습니다.** Cline은 GitHub에서 직접 리포지토리를 복제하고 서버를 빌드할 수 있습니다.
-   **GitHub:** GitHub에서 MCP 서버를 찾는 가장 일반적인 두 곳은 다음과 같습니다.
    -   [공식 MCP 서버 리포지토리](https://github.com/modelcontextprotocol/servers)
    -   [Awesome-MCP 서버 리포지토리](https://github.com/punkpeye/awesome-mcp-servers)
-   **온라인 디렉터리:** 다음을 포함하여 여러 웹사이트에 MCP 서버가 나열되어 있습니다.

    -   [mcpservers.org](https://mcpservers.org/)
    -   [mcp.so](https://mcp.so/)
    -   [glama.ai/mcp/servers](https://glama.ai/mcp/servers)

    이러한 디렉터리를 통해 사용자는 다운로드, 날짜, 별표 및 사용 사례와 같은 다양한 기준으로 서버를 정렬할 수 있습니다. 각 항목은 기능, 도구 및 구성 지침과 같은 정보를 제공합니다.

-   **PulseMCP:** 이 웹사이트에는 AI가 MCP 서버를 사용하여 웹사이트를 쓸모없게 만드는 방법에 대해 논의하는 블로그 게시물이 있습니다. PulseMCP에는 MCP 서버에 대한 FAQ 섹션도 포함되어 있습니다. [https://www.pulsemcp.com/](https://www.pulsemcp.com/)

## **Cline으로 빌드**

1. **프로세스 시작:** Cline에 다음 정보를 제공합니다.
    - **GitHub 리포지토리 URL:** 서버 리포지토리의 URL입니다.
    - **README.md 내용:** 선택 사항이지만 Cline이 서버의 목적과 구성을 이해하는 데 도움이 됩니다. GitHub 리포지토리에서 README.md 파일을 복사할 수 있습니다.
2. **Cline과의 상호 작용 예시:**

    ```
    사용자: "Cline, Brave 브라우저 제어를 위한 MCP 서버를 추가하고 싶습니다. GitHub 링크는 다음과 같습니다. @https://github.com/modelcontextprotocol/servers/tree/main/src/brave-search 추가해 주시겠어요?"

    Cline: "알겠습니다. 리포지토리를 MCP 디렉터리로 복제합니다. 'package.json' 파일이 있으므로 빌드해야 합니다. 'npm run build'를 실행해야 할까요?"

    사용자: "예, 빌드하세요."

    Cline: "빌드가 완료되었습니다. 이 서버에는 브라우저 경로가 필요합니다. Brave 브라우저 실행 파일은 어디에 있습니까?"

    사용자: " '/Applications/Brave Browser.app/Contents/MacOS/Brave Browser'에 있습니다."

    Cline: "알겠습니다. 해당 경로로 서버 구성을 업데이트하고 'cline_mcp_settings.json'에 추가했습니다. 다른 것이 있습니까?"

    사용자: "아니요, 그게 다입니다. 테스트해 봅시다."

    Cline: "좋습니다! 서버 연결을 테스트하기 위해 MCP 검사기를 시작합니다. 그런 다음 Cline에서 브라우저를 제어해 볼 수 있습니다."
    ```

3. **Cline의 작업:** 지침에 따라 Cline은 다음을 수행합니다.
    - **리포지토리 복제:** Cline은 일반적으로 구성에 지정된 디렉터리에 리포지토리를 로컬 컴퓨터로 복제합니다.
    - **조정:** Cline이 서버 구성을 수정하도록 안내할 수 있습니다. 예를 들어:
        - **사용자:** "이 서버에는 API 키가 필요합니다. 어디에 추가해야 하는지 찾아주시겠어요?"
        - Cline은 지침에 따라 `cline_mcp_settings.json` 파일 또는 기타 관련 파일을 자동으로 업데이트할 수 있습니다.
    - **서버 빌드:** Cline은 서버에 적합한 빌드 명령(일반적으로 `npm run build`)을 실행합니다.
    - **설정에 서버 추가:** Cline은 서버 구성을 `cline_mcp_settings.json` 파일에 추가합니다.

## **테스트 및 문제 해결**

1. **서버 테스트:** Cline이 빌드 프로세스를 완료하면 서버가 예상대로 작동하는지 테스트합니다. 문제가 발생하면 Cline이 도움을 줄 수 있습니다.
2. **MCP 검사기:** MCP 검사기를 사용하여 서버의 연결 및 기능을 테스트할 수 있습니다.

## **모범 사례**

-   **기본 사항 이해:** Cline은 프로세스를 단순화하지만 서버 코드, MCP 프로토콜([자세히 알아보기](/mcp/mcp-overview)) 및 서버 구성 방법에 대한 기본적인 이해가 있으면 도움이 됩니다. 이를 통해 더 효과적인 문제 해결 및 사용자 지정이 가능합니다.
-   **명확한 지침:** 프로세스 전반에 걸쳐 Cline에 명확하고 구체적인 지침을 제공합니다.
-   **테스트:** 설치 및 구성 후 서버를 철저히 테스트하여 올바르게 작동하는지 확인합니다.
-   **버전 관리:** 버전 관리 시스템(예: Git)을 사용하여 서버 코드 변경 사항을 추적합니다.
-   **최신 상태 유지:** 최신 기능 및 보안 패치의 이점을 누리려면 MCP 서버를 최신 상태로 유지하십시오.
